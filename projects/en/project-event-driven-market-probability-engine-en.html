Personal Project · In Progress

Event-Driven Market Probability Engine

A quantitative research platform that estimates next-day market move probabilities around macro events using a reproducible data + modeling pipeline.

Back to portfolio

Summary

End-to-end pipeline that ingests historical market data and event metadata, engineers statistical features, trains a probabilistic model, and visualizes signal quality and strategy outcomes in Power BI.

Architecture, data engineering, modeling, visualization
PostgreSQL, Python (NumPy/Pandas, optional scikit-learn), dbt, Power BI, Git
In progress (MVP pipeline + first dashboard in development)

Coming soon

Features

Data Ingestion

Market prices + event metadata loaded into PostgreSQL with a clear raw/staging/analytics separation.

Feature Engineering

Statistical features (returns, rolling volatility, momentum, event “surprise” proxies) built in Python (NumPy/Pandas).

Probabilistic Modeling

Baseline probability model (logistic regression), extensible to tree-based methods; outputs calibrated probabilities.

dbt Transformations

Reproducible SQL models and tests (uniqueness, not-null, relationships) producing analytics-ready fact/dimension tables.

Backtesting & Metrics

Offline signal evaluation: ROC/AUC, calibration checks, and strategy performance metrics (PnL, drawdown).

Power BI Dashboard

Interactive pages for signal quality, event segmentation, and risk/performance analysis over time.

Architecture

Data & Modeling Pipeline

Raw market/event datasets are stored in PostgreSQL, enriched with Python feature engineering, and transformed via dbt into a star schema. Model outputs (probabilities, metrics) are written back to the database for reproducible analysis.

Analytics & Reporting

Power BI connects to the analytics schema to visualize probability distributions, event-conditioned performance, and backtest results.
