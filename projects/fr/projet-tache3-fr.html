IFT3913

GitHub Actions (Quality Gate)

Pipeline CI basé sur PIT : comparaison avec une baseline, échec automatique sur régression, et rapports accessibles via artifacts. Réalisé lors de mon cours de qualité logiciel (IFT3913).

Repo GitHub Résumé

Résumé

Mise en place d’un “gate” de qualité dans GitHub Actions : le build échoue si le score PIT diminue. Une baseline stable est conservée et mise à jour uniquement quand master est “green”.

Fail on mutation score regression
Cache GitHub Actions (référence master)
Upload artifact des rapports PIT

On failure

Validation (scénarios)

Baseline (succès sur master)

Quand le build passe sur master, la baseline est (re)générée et sauvegardée, garantissant une référence fiable pour les exécutions suivantes.

Régression (échec sur branche)

Sur une branche, si une régression PIT est introduite, l’étape de comparaison échoue et empêche la validation du pipeline.
